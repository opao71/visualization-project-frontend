# 快速启动指南 - 性能优化版

## 🚀 立即体验优化效果

### 1. 启动项目
```bash
cd visualization-project-frontend
npm run serve
```

或直接双击：`start.bat`

### 2. 访问优化版页面
打开浏览器访问：**http://localhost:8080/summit**

### 3. 观察优化效果

#### 首屏加载
- ✅ 立即显示 Hero 部分（<1秒）
- ✅ 显示"正在加载数据..."遮罩
- ✅ 加载完成后遮罩消失

#### 滚动加载
- ✅ 向下滚动时，图表才开始加载
- ✅ 显示骨架屏占位
- ✅ 数据加载完成后平滑显示

#### 性能对比
打开浏览器开发者工具（F12）：
- **Network 标签**：观察请求时机
- **Performance 标签**：录制加载过程

## 🎯 优化关键点

### 1. 懒加载机制
```
用户滚动 → 图表进入视口 → 开始加载数据 → 显示图表
```

### 2. 加载状态
- **全局遮罩**：页面初始化时
- **骨架屏**：图表未加载时
- **加载提示**：数据请求中

### 3. 性能提升
- 首屏加载时间：**3-5s → 0.5-1s**
- 初始请求数量：**9+ → 2**
- 用户可交互时间：**4-6s → 1s**

## 🔧 如果遇到问题

### 问题1：页面空白
**原因**：后端服务未启动

**解决**：
```bash
cd visualization-project-backend
python app.py
```

### 问题2：图表不显示
**检查**：
1. 打开浏览器控制台（F12）
2. 查看 Network 标签是否有红色请求
3. 确认后端 API 地址：`http://localhost:5000`

### 问题3：加载仍然慢
**可能原因**：
- 后端数据量大
- 网络延迟
- 数据库查询慢

**优化建议**：
- 添加后端缓存
- 优化数据库查询
- 减少返回数据量

## ✨ 核心优化技术

1. **IntersectionObserver** - 监听元素进入视口
2. **懒加载组件** - 按需渲染
3. **骨架屏** - 提升感知性能
4. **异步加载** - 并行请求优化
